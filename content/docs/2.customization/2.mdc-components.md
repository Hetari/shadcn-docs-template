# MDC Components

This document covers the three main custom MDC (Markdown Components) used in this template to seamlessly display UI component previews and their raw source code.

## ComponentPreview

The `<ComponentPreview>` (or `::component-preview` in Markdown) is the main component used to showcase UI components and their examples. It automatically renders a working interactive component alongside its source code in a tabbed interface.

### File Location
`app/components/content/ComponentPreview.vue`

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `name` | `string` | *(Required)* | The name of the component (e.g., `Button`). It will look for the component first in `components/demo/`, and if not found, it falls back to `components/ui/`. |
| `align` | `"center" \| "start" \| "end"` | `"center"` | Alignment of the rendered component inside the preview box. |
| `description` | `string` | `undefined` | An optional description for the component. |
| `hideCode` | `boolean` | `false` | If `true`, the "Code" tab and source code will be completely hidden, rendering only the interactive component block. |
| `class` | `HTMLAttributes["class"]` | `undefined` | Custom CSS classes applied to the root wrapper. |

### Usage

```md
::component-preview
---
name: Button
description: A basic button example
align: start
---
::
```

Or using the shorthand inline syntax:

```md
::component-preview{name="Button" align="end"}
```

## ComponentSource

The `<ComponentSource>` (or `::component-source` in Markdown) component fetches and displays the formatted raw source code of a specified component. It's used internally by `ComponentPreview` but can be used standalone when you only want to display a code file block.

### File Location
`app/components/content/ComponentSource.vue`

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `name` | `string` | `undefined` | The name of the component to fetch the source for (looks in `components/demo` then `components/ui`). |
| `title` | `string` | `undefined` | A title or filepath displayed at the top of the code block. |
| `language` | `string` | `"vue"` | The programming language for syntax highlighting. |
| `collapsible` | `boolean` | `true` | If `true`, wraps the code block in `<CodeCollapsibleWrapper>` to save vertical space. |
| `class` | `HTMLAttributes["class"]` | `undefined` | Custom CSS classes. |
| `chromeLessOnMobile`| `boolean` | `undefined` | Flag to simplify the code block display on smaller screens. |

### Usage

```md
::component-source
---
name: Input
title: components/ui/input/Input.vue
collapsible: false
language: vue
---
::
```

## CodeCollapsibleWrapper

The `<CodeCollapsibleWrapper>` (or `::code-collapsible-wrapper` in Markdown) is a utility wrapper component that limits the initial height of large blocks of content (such as long source code blocks) and provides an "Expand" / "Collapse" toggle button to view the full content.

### File Location
`app/components/content/CodeCollapsibleWrapper.vue`

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `class` | `HTMLAttributes["class"]` | `undefined` | Custom CSS classes for the wrapper. |

### Usage

You can wrap any large content, commonly code blocks, with this component:

```md
::code-collapsible-wrapper
```vue
<template>
  <div>
    <!-- Very long code snippet here... -->
  </div>
</template>
```
::
```

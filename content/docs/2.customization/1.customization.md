This guide walks you through every touchpoint you need to personalize this template — from navigation links to the command menu search palette.

## Site Configuration

The central place for branding and links is `app/lib/siteConfig.ts`. Update these values **first**:

```ts
// app/lib/siteConfig.ts
export const siteConfig: SiteConfig = {
  name: "My Project", // shown in header & OG tags
  url: "https://my-project.dev",
  ogImage: "https://my-project.dev/og.jpg", // social sharing image
  description: "A short description of your project.",
  links: {
    github: "https://github.com/you/my-project",
    githubAPI: "https://api.github.com/repos/you/my-project",
    ungh: "https://ungh.cc/repos/you/my-project",
  },
  navItems: [
    // These items appear in both desktop and mobile navigation
    { href: "/docs/getting-started/introduction", label: "Docs" },
    { href: "/blog", label: "Blog" },
  ],
};
```

The `navItems` array is the **single source of truth** for the top-level navigation on both desktop and mobile. Every component reads from it automatically.

## Desktop Navigation — `MainNav.vue`

**File:** `app/components/MainNav.vue`

`MainNav.vue` receives its items via a prop; you do **not** need to edit this component to change routes.

### Changing Routes

1. Open `app/lib/siteConfig.ts`.
2. Edit the `navItems` array — add, remove, or reorder items:

```ts
navItems: [
  { href: "/docs/getting-started/introduction", label: "Docs" },
  { href: "/showcase", label: "Showcase" }, // new link
  { href: "/pricing", label: "Pricing" }, // new link
],
```

3. Save the file. The desktop nav updates automatically.

### How It Works

`SiteHeader.vue` passes `siteConfig.navItems` directly to `MainNav.vue`:

```vue
<MainNav :items="siteConfig.navItems" class="hidden lg:flex" />
```

`MainNav.vue` iterates over the `items` prop and highlights the link whose `href` matches the current route:

```vue
<NuxtLink
  :class="path === item.href ? 'text-primary' : 'text-muted-foreground'"
  :to="item.href"
>
  {{ item.label }}
</NuxtLink>
```

## Mobile Navigation — `MobileNav.vue`

**File:** `app/components/MobileNav.vue`

The mobile menu combines **three** sources of links:

| Section                                          | Source                         | How to Change                |
| ------------------------------------------------ | ------------------------------ | ---------------------------- |
| **Main Links** (Home, Docs, …)                   | `siteConfig.navItems`          | Edit `app/lib/siteConfig.ts` |
| **Sections** (Getting Started, Configuration, …) | `NAV_SECTIONS`                 | Edit `app/lib/navigation.ts` |
| **Content Tree** (auto-generated sidebar)        | `content/` directory structure | Add/remove markdown files    |

### Changing Main Links

Same as desktop — update `siteConfig.navItems` in `app/lib/siteConfig.ts`. The mobile nav receives them as the `items` prop:

```vue
<MobileNav :tree="data ?? []" :items="siteConfig.navItems" />
```

### Changing Sections

The "Sections" block is driven by three constants in `app/lib/navigation.ts`:

```ts
// app/lib/navigation.ts

/** Top-level section pills shown in the sidebar and mobile menu. */
export const NAV_SECTIONS: NavItem[] = [
  { name: "Get Started", href: "/docs/introduction" },
  { name: "Components", href: "/docs/components" },
  { name: "Registry", href: "/docs/registry" },
  { name: "MCP Server", href: "/docs/mcp" },
  { name: "Forms", href: "/docs/forms" },
  { name: "Changelog", href: "/docs/changelog" },
];

/** Sidebar group titles (lowercased) that should never be rendered. */
export const SIDEBAR_EXCLUDED_SECTIONS: string[] = ["installation", "dark mode"];

/** Individual page paths that should be hidden from the sidebar page list. */
export const SIDEBAR_EXCLUDED_PAGES: string[] = ["/docs/introduction", "/docs/changelog"];
```

| Constant                    | What it controls                                      |
| --------------------------- | ----------------------------------------------------- |
| `NAV_SECTIONS`              | Section pills in both desktop sidebar and mobile menu |
| `SIDEBAR_EXCLUDED_SECTIONS` | Sidebar group titles (lowercased) to hide completely  |
| `SIDEBAR_EXCLUDED_PAGES`    | Individual page paths to hide from the sidebar        |

### Changing the Content Tree

The bottom part of the mobile menu renders the documentation sidebar automatically from your `content/` directory. To change it:

1. **Add a page:** Create a new `.md` file in `content/docs/`.
2. **Create a group:** Create a numbered subdirectory (e.g., `content/docs/2.guides/`).
3. **Control ordering:** Prefix filenames and folders with numbers (`1.introduction.md`, `2.setup.md`).
4. **Customize navigation metadata:** Add a `.navigation.yml` file to any directory:

```yaml
# content/docs/2.guides/.navigation.yml
title: Guides
icon: lucide:book-open
```

### Feature Flags

The mobile nav has a feature flag `showMcpDocs` (from `app/lib/flag.ts`) that controls the visibility of MCP-related links. Set it to `false` to hide them:

```ts
// app/lib/flag.ts
export const showMcpDocs = false;
```

## Command Menu Search — `CommandMenu.vue`

**File:** `app/components/CommandMenu.vue`

The command palette opens via :kbd[⌘ K] (Mac) or :kbd[Ctrl K] (Windows/Linux) and provides instant search over pages, colors, and blocks.

### How Search Works

The command menu uses a custom filter function that matches against **both** the item value and its keywords:

```ts
:filter="(value, search, keywords?) => {
  const extendValue = `${value} ${keywords?.join(' ') || ''}`
  if (extendValue.toLowerCase().includes(search.toLowerCase())) {
    return 1
  }
  return 0
}"
```

Each `CommandItem` or `CommandMenuItem` accepts:

- **`value`** — the primary string matched against the search query.
- **`keywords`** — an optional array of extra search terms (e.g., `['color', 'blue', 'blue-500']`).

### Adding Custom Items to Search

To add a new group of searchable items:

1. Add a new prop or data source to the `Props` interface:

```ts
type Props = {
  tree?: { ... };
  colors: ColorPalette[];
  blocks?: { name: string; description: string; categories: string[] }[];
  navItems?: { href: string; label: string }[];
  // Add your custom items:
  tools?: { name: string; url: string; category: string }[];
};
```

2. Add a new `<CommandGroup>` inside `<CommandList>`:

```vue
<CommandGroup
  v-if="tools?.length"
  heading="Tools"
  class="p-0! **:data-[slot=command-group-heading]:p-3!"
>
  <CommandMenuItem
    v-for="tool in tools"
    :key="tool.name"
    :value="tool.name"
    :keywords="['tool', tool.category]"
    @select="() => runCommand(() => navigateTo(tool.url, { external: true }))"
  >
    {{ tool.name }}
    <span class="ml-auto text-xs text-muted-foreground">
      {{ tool.category }}
    </span>
  </CommandMenuItem>
</CommandGroup>
```

3. Pass the data from `SiteHeader.vue`:

```vue
<CommandMenu
  :tree="docData"
  :colors="getColors()"
  :nav-items="siteConfig.navItems"
  :tools="myTools"
/>
```

### Removing the Colors Section

If you don't need the Tailwind color palette in search:

1. **Delete the color group template** — remove the entire `<CommandGroup v-for="colorPalette in colors" ...>` block (around lines 180-203).
2. **Remove the color prop** — delete `colors: ColorPalette[]` from the `Props` type.
3. **Clean up imports** — remove the `Color` and `ColorPalette` type imports.
4. **Update `SiteHeader.vue`** — remove `:colors="getColors()"` from the `<CommandMenu>` tag and the `getColors` import.
5. **Remove the footer hint** — in the bottom bar, remove or modify the `<span v-if="selectedType === 'color'">Copy OKLCH</span>` line.
6. **Delete the handler** — remove the `handleColorHighlight` function.

### Removing the Blocks Section

Similarly, to remove blocks from the command menu:

1. Delete the `<CommandGroup v-if="blocks?.length" ...>` block (around lines 204-223).
2. Remove `blocks` from the `Props` type.
3. Remove the `handleBlockHighlight` function.

### Customizing Keyboard Shortcuts

The command menu opens on :kbd[⌘ K] / :kbd[Ctrl K] and also on :kbd[/]. To change this, edit the `down` function inside `onMounted`:

```ts
function down(e: KeyboardEvent) {
  // Change the trigger key here:
  if (e.key === "k" && (e.metaKey || e.ctrlKey)) {
    // ...
    e.preventDefault();
    open.value = !open.value;
  }
}
```

## Styling the Command Items — `CommandMenuItem.vue`

**File:** `app/components/CommandMenuItem.vue`

This component wraps the base `<CommandItem>` from shadcn-vue with custom styling and a highlight-detection mechanism via `MutationObserver`.

### Changing Item Appearance

Edit the class string to update how items look on hover/selection:

```vue
<CommandItem
  :class="cn(`
    h-9 rounded-md border border-transparent px-3! font-medium
    data-highlighted:border-input data-highlighted:bg-input/50
    data-[selected=true]:border-input data-[selected=true]:bg-input/50
  `, props.class)"
>
```

| What to Change  | Edit                                                |
| --------------- | --------------------------------------------------- |
| Row height      | `h-9` → `h-10`, `h-12`, etc.                        |
| Border radius   | `rounded-md` → `rounded-lg`, `rounded-full`         |
| Highlight style | `data-highlighted:bg-input/50` → any Tailwind class |
| Padding         | `px-3!` → your preferred padding                    |

### The `@highlight` Event

`CommandMenuItem` emits a `highlight` event when the user hovers over or keyboard-navigates to the item. The parent `CommandMenu.vue` uses this to update the status bar at the bottom of the dialog (e.g., showing "Copy OKLCH" or a `shadcn-vue add` command).

## Keyboard Shortcut Badges — `CommandMenuKbd.vue`

**File:** `app/components/CommandMenuKbd.vue`

This is a simple presentational component that renders `<kbd>` elements in the command menu footer.

### Changing Badge Styling

Edit the class list on the `<kbd>` element:

```vue
<kbd
  class="
    pointer-events-none flex h-5 items-center justify-center gap-1
    rounded-sm border bg-background px-1
    font-sans text-[0.7rem] font-medium text-muted-foreground select-none
  "
>
  <slot />
</kbd>
```

| What to Change | Edit                         |
| -------------- | ---------------------------- |
| Badge height   | `h-5` → `h-6`                |
| Border radius  | `rounded-sm` → `rounded-md`  |
| Background     | `bg-background` → `bg-muted` |
| Font size      | `text-[0.7rem]` → `text-xs`  |
